# Premium Ebook Master Add-On

This is the **non-negotiable quality contract** for every ebook the factory ships. If any rule fails, the pipeline must auto-fix and re-QC — never upload to Shopify with a failing score.

## 1. Real-book page layout

Every chapter body page must read like a printed nonfiction book, not a pasted article.

- Chapter title: left-aligned, bold, max 2–3 lines, generous space below (≥ 20pt), no border-crammed title block.
- Keep title with first paragraph (`page-break-after: avoid`) — never orphan a heading.
- Balanced top / left / right margins; body never touches page edges.
- Chapter opener never opens with a wall of text — split into 2–3 paragraphs on natural transitions.
- Enforced in `supabase/functions/_shared/pdf-template.ts` via `.chapter-body__title` and `splitLongParagraphs()`.

## 2. Justified premium body text

- `text-align: justify`, `hyphens: auto`, `hyphenate-limit-chars: 6 3 3`.
- `line-height: 1.6`, `max-width: 5.4in` — no overly wide columns, no ragged right.
- Any `<p>` over ~140 words is auto-split on sentence boundaries by `splitLongParagraphs()`.
- Drop cap on first paragraph of each chapter.
- Repeating `<thead>` for long tables (`display: table-header-group`).
- QC gate: `typography_score`, `reading_comfort_score`, `premium_layout_score` all ≥ 90 (`formatterGate` in `render-pdf`).

## 3. Natural-language reader QC

- `reader-experience-qc` edge function is a **mandatory gate** between `manuscript_qc` and `cover`.
- Deterministic screens catch AI patterns, repetitive structure, templated phrasing.
- AI critic scores 11 dimensions (naturalness, page-turning, emotional resonance, sellability, …).
- Pass targets ≥ 90. Failing scores trigger the Targeted Humanize Loop (max 3 attempts) that rewrites only flagged excerpts while preserving chapter integrity.

## 4. PDF cover fills a true A4 page

- Document geometry is `A4 portrait` end-to-end in `pdf-template.ts`.
- Cover page uses `.cover-a4` with `@page cover-a4 { size: A4; margin: 0 }` — zero margins, zero borders, full bleed.
- QC gate: `pdf_cover_full_a4_score` must equal **100**. Anything less blocks the render and re-runs cover generation.
- AI image is textless background only. Title / subtitle / brand are HTML/SVG typographic overlays.

## 5. Thumbnail is a real book mockup

- Generated by `renderThumbnail()` in `generate-cover/index.ts`.
- Perspective-skewed front cover, visible spine, gradients, sheen, radial ground shadow — never a flat crop of the PDF page.
- QC gates (all ≥ 90): `thumbnail_book_mockup_score`, `thumbnail_readability_score`, `shopify_click_appeal`, `premium_product_feel`.

## 6. Auto-fix before Shopify — no exceptions

Every QC gate above runs **before** `shopify_draft`. If any score fails:

1. Pipeline classifies the failure via `_shared/error-classifier.ts`.
2. Runs the matching targeted repair loop (max 3 attempts):
   - Formatting → re-render PDF with adjusted template hints.
   - Reader QC → Targeted Humanize Loop on flagged excerpts.
   - Cover / thumbnail → rebuild with repair hints (`repair_hint` on generation prompt).
3. Re-runs QC after each attempt.
4. Only when **all** gates pass does the run advance to `shopify_draft` → `shopify_verify` → `final_report`.
5. If 3 attempts fail, mark run `needs_admin` with structured `failed_reasons` — never silently upload a failing ebook.

## Enforcement checklist (apply to every ebook)

- [ ] `formatterGate` passed (typography, reading comfort, table, worksheet, premium layout ≥ 90)
- [ ] `raw_markdown_score` = 100 (no pipe tables, no stray `#`, `*`)
- [ ] `reader_experience_qc` passed (all 11 scores ≥ 90)
- [ ] `pdf_cover_full_a4_score` = 100
- [ ] `thumbnail_book_mockup_score` ≥ 90, `premium_product_feel` ≥ 90
- [ ] All repair loops exhausted before any `needs_admin` status
- [ ] Shopify upload only after every gate above is green
